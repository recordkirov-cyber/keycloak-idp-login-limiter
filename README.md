# Keycloak IdP Login Limiter Condition

Кастомный **Condition Authenticator** для Keycloak, который ограничивает количество аутентификаций пользователя через указанный Identity Provider (IdP) в пределах заданного интервала.

## 🔎 Описание
Этот аутентификатор проверяет количество попыток входа пользователя через конкретный провайдер IdP и возвращает `false`, если лимит превышен. Это позволяет использовать ветку `Alternative` в потоке аутентификации, чтобы показать сообщение об ошибке, перенаправить пользователя или прервать вход.

## 🚀 Функциональные возможности
- Ограничение количества аутентификаций по IdP
- Поддержка глобального лимита (если IdP не указан)
- Персональные пользовательские атрибуты для хранения счётчиков
- Автоматический сброс счётчика по времени (интервал в часах)
- Отображение оставшихся попыток через атрибут пользователя
- Можно сбросить счётчики вручную в админ-консоли пользователя
- Работает с Keycloak 22+ и Java 21+

## 📦 Требования
- **Keycloak**: 22.0.0+
- **Java**: 21+
- **Maven**: 3.8+

## 🛠 Сборка
```bash
mvn clean package
```

После успешной сборки JAR появится в `target/keycloak-idp-login-limiter-${version}.jar`.

## 📥 Установка
1. Скопируйте собранный JAR в директорию `providers` Keycloak или в другую директорию провайдеров в зависимости от конфигурации сервера.
2. Перезапустите Keycloak.
3. В админ-консоли создайте или отредактируйте поток аутентификации.
4. Добавьте новый `Condition - Identity Provider Rate Limiting`.

## ⚙️ Конфигурация аутентификатора
В настройках аутентификатора доступны следующие параметры:

| Поле | Имя конфигурации | Описание | Пример |
|---|---|---|---|
| Лимит входов | `idp-limit` | Максимальное число аутентификаций через указанный IdP за интервал | `5` |
| Алиас IdP | `idp-alias` | Алиас провайдера, для которого считается лимит. Если пусто — глобальный счётчик для всех IdP | `google` |
| Интервал сброса | `reset-interval-hours` | Интервал в часах, после которого автоматически сбрасывается счётчик | `24` |

### Примеры конфигурации
- `idp-limit = 5`, `idp-alias = google`, `reset-interval-hours = 24` — максимум 5 входов через `google` в сутки.
- `idp-limit = 10`, `idp-alias = `, `reset-interval-hours = 12` — максимум 10 входов через любой IdP за 12 часов.

## 🧩 Как это работает
1. Извлекается пользователь из контекста аутентификации.
2. Определяется IdP:
   - если указан `idp-alias`, используется он;
   - иначе проверяются заметки сессии `BROKER_IDENTITY_PROVIDER` и `IDENTITY_PROVIDER`;
   - если IdP не найден — условие не выполняется.
3. Проверяется время последнего сброса атрибута.
4. Если прошло больше заданного числа часов, счётчик сбрасывается.
5. Счётчик инкрементируется и сравнивается с лимитом.
6. Обновляется атрибут оставшихся попыток.

## 🧾 Пользовательские атрибуты
Для каждого пользователя используются атрибуты вида:
- `idp_attempts_{alias}` — текущее число входов через IdP
- `idp_last_reset_{alias}` — метка времени последнего сброса
- `idp_remaining_attempts_{alias}` — оставшееся число попыток

Если алиас не указан, используется суффикс `global`:
- `idp_attempts_global`
- `idp_last_reset_global`
- `idp_remaining_attempts_global`

### Примечание
Атрибут `idp_remaining_attempts_*` обновляется автоматически после каждой попытки входа и показывает, сколько входов осталось до достижения лимита.

## 🔧 Сброс в админ-консоли
Администратор может вручную сбросить счётчики у пользователя, удалив или изменив соответствующие атрибуты в профиле пользователя.

### Где искать
В админ-консоли Keycloak:
- `Users` → выберите пользователя → вкладка `Attributes`
- Найдите `idp_attempts_*`, `idp_last_reset_*`, `idp_remaining_attempts_*`
- Удалите или измените значения, чтобы сбросить счётчики

## 📌 Замечания
- При отсутствии корректной конфигурации лимита аутентификации плагин выберет безопасное поведение и не заблокирует пользователя.
- Сброс по времени работает на уровне часов и задаётся администратором через параметр `reset-interval-hours`.
