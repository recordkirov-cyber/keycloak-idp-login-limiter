# Keycloak IdP Login Limiter Condition

Кастомный **Condition Authenticator** для Keycloak 22+, который ограничивает количество успешных аутентификаций через конкретный Identity Provider (IdP). При достижении лимита возвращает `false`, позволяя перенаправить пользователя на ветку `Alternative` для показа сообщения об ошибке или редиректа.

## 🚀 Возможности
- ✅ Фильтрация по конкретному IdP (алиас)
- ✅ Настраиваемый лимит входов
- ✅ Хранение счётчика в пользовательском атрибуте
- ✅ Безопасное обновление атрибута в рамках транзакции аутентификации
- ✅ Логирование через JBoss Logging
- ✅ Fail-safe логика (при ошибках конфигурации не блокирует пользователя)

## 📦 Требования
- **Keycloak**: 22.0.0+ (Jakarta EE, Java 17)
- **Maven**: 3.8+
- **Java**: 21+
